# O-CONNECT · Приемане на стока (EAN reconciliation)

Уеб инструмент за склада, който решава забавянето при приемане, когато доставчикът праща
**верния продукт, но с различен баркод (EAN)** от поръчания.

## Модел на данните (важно)

- **Голям файл „Поръчки на стока“ = source of truth.** Оттук идва EAN-ът, който трябва да се
  залепи в системата, и датата на поръчката. Един EAN на ред, по доставчик, по дни (листове).
- **Таблица на Иван = речник на вариантите.** За всеки продукт изброява кои баркодове са еднакви
  (доставчикът понякога праща различен от поръчания).

При приемане приемчикът сканира физическия баркод:
- 🟢 **Сканираният = поръчаният EAN** (от големия файл) → нищо за правене.
- 🟠 **Вариант** → попъп: **„Залепи EAN ___, поръчка от ___ , доставчик ___“** (взето от големия
  файл) + бутон „Копирай“.
- 🔴 **Непознат / без поръчан EAN** → свързва се веднъж с продукт и се помни занапред.

## Какво качва складът за всяка доставка

1. **Голям файл „Поръчки на стока“ (Excel) — задължителен.** После избира **доставчика**.
   Това е истината за поръчания EAN и датата.
2. **Таблица на Иван (Excel) — препоръчителна.** Дефинира продуктите в доставката и вариантите.
   Без нея инструментът работи в „само голям файл“ режим (точно съвпадение + учи вариантите).
3. **Опаковъчен лист (PDF) — по избор.** Добавя номер на кашон.

Историята на доставките и научените EAN-и се пазят локално в браузъра на устройството.

## Пускане на публичен сайт (GitHub Pages)

1. GitHub (`LYUBOOOO`) → **New repository** → напр. `priemane` → Public → Create.
2. **Add file → Upload files** → пусни всички файлове (със запазена папка `vendor/`) → Commit.
3. **Settings → Pages** → Deploy from a branch → `main`, `/ (root)` → Save.
4. Сайтът е на `https://LYUBOOOO.github.io/priemane/` — дай линка на склада.

## Структура

```
index.html                 ← приложението
vendor/xlsx.full.min.js    ← четец на Excel (SheetJS, локален)
.nojekyll
README.md
```

Без бекенд. Excel се чете в браузъра. Големият файл се обработва при всяка доставка (затова е
винаги актуален). Историята е per устройство; за споделена история между станции трябва бекенд
(може на Render/DigitalOcean) — следваща стъпка при нужда.
